## Data Flow

1. **Guest Visit**: Landing page loads with price data and settings from the System Config (Supabase).
2. **Date Selection**: Guest selects dates via `CoastalAvailability`. Prices are calculated client-side based on the retrieved base rate.
3. **Request Submission**: Guest fills the recommendation-based form. Data is sent to the Route Handler.
4. **Validation & Notification**: The API validates the payload with Zod, records the request in Supabase, and dispatches a notification email via Resend to the owner.
